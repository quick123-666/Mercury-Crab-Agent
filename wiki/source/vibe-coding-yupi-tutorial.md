# Vibe Coding 零基础入门教程

## Summary

本文档来自全网200万粉丝的AI编程博主鱼皮（Code Father）的BiliBili视频专栏，系统性介绍Vibe Coding（意图驱动的AI辅助编程）这一新兴编程范式。核心观点：2026年，开发产品的规则已经彻底改变——AI让编程从"写代码"变成"写需求"。

## Key Claims & Facts

**Claims:**
- 2026年，开发产品的规则已经彻底改变——AI让编程从'写代码'变成'写需求' (confidence: high)
- Vibe Coding = 用自然语言和AI聊天 + AI帮你写代码 + 一起迭代优化 (confidence: high)
- 在Vibe Coding时代，最重要的'编程语言'不是Python、JavaScript，而是你的母语 (confidence: high)
- AI不是工具，而是编程伙伴 (confidence: high)
- 天气查询应用传统编程需要几周，Vibe Coding半小时搞定 (confidence: medium)
- 学生周报生成器项目从开始到完成花了不超过10分钟，全部由AI生成代码 (confidence: high)
- 《编程导航》小程序用Vibe Coding在一周时间内开发完成上线 (confidence: high)
- Vibe Coding存在边界同质化问题（蓝紫渐变） (confidence: high)
- AI生成代码存在不可控风险，复杂项目调试困难 (confidence: high)
- 长期使用Vibe Coding可能导致编程技能退化 (confidence: high)
- Claude Code目前仅支持Linux和MacOS，Windows需要WSL (confidence: high)
- 鱼皮全网200万粉丝 (confidence: high)

**Facts:**
- Vibe Coding定义：以自然语言提示驱动LLM，由AI直接生成并迭代代码的意图驱动型开发模式
- 框架转变：从关注'怎么做'(How)转向关注'做什么'(What)
- AI角色转变：从工具转变为编程伙伴（产品经理+工程师协作模式）
- 传统编程 vs Vibe Coding：语法记忆→需求表达；陡峭学习曲线→几天上手
- 应用范围：网页应用、小程序、AI应用、数据处理工具、自动化脚本、辅助工具
- 三大误区：①不是复制粘贴 ②不会失去学习能力 ③能做复杂项目
- 当前局限：边界同质化、代码不可控风险、技能退化风险
- 零基础学习路径：第1周读完全文+快���上手+零代码平台；第2-3周学Cursor+做项目
- 鱼皮团队编程导航小程序：用Vibe Coding一周开发完成

## Contradictions & Gaps

**Contradictions:**
- 文档声称"2026年开发产品规则彻底改变"，但wiki中Claude Code相关文档显示其功能（如MCP、subagents等高级功能）需要较深的技术理解，与"几天就 能上手"的说法存在张力
- 文档宣称"Vibe Coding可以做几乎所有软件开发"，但同时承认存在"代码不可控"和"技能退化"的风险，略现矛盾

**Data Gaps:**
- 文档未提供Cursor等工具的具体使用教程细节
- 未说明Vibe Coding在不同代码代币库下的能力表现数据
- 缺乏与其他AI编程工具（如Copilot、Claude Code）的对比分析
- 学习路径中提到的"零代码平台"具体指哪些工具未列出
- Claude Code相关内容（wiki中有大量记录）在本文中未提及，但两者高度相关
