## 00:52 — 用户两个核心架构问题

### 问题1：目录创建如何避免重名

**方案：两层目录结构**
```
memory/
├── <qq_id>/                    ← 第一层：用户身份（QQ sender_id）
│   ├── CLAUDE_CODE/            ← 第二层：程序身份
│   │   ├── MEMORY.md
│   │   ├── preferences.md
│   │   └── conversations/
│   ├── CURSOR/
│   └── WINDSURF/
├── user_B/
└── shared/                     ← 公共知识
```

**避免重名机制**：
- 第一层用 QQ sender_id（全球唯一）
- 第二层用程序上报的固定名称（不重复）

### 问题2：程序丢失记忆后如何找到账户

三种方案对比：

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **A: Key-File** | 注册时分配 API Key，存本地文件 | 换电脑也能找回 | 需要安全存储 Key |
| **B: QQ号+程序名** | 目录 = qq_id/program_name | 简单，无需 Key | 多实例会冲突 |
| **C: 混合方案** | Key 做身份验证，目录用 program_name | 最稳妥 | 实现复杂度中等 |

**推荐方案 C（混合）**：
- 注册时分配唯一 Key → 写入 `~/.mercury-crab/credential`
- 目录结构 → `memory/<qq_id>/<program_name>/`
- 程序重连时用 Key 验证身份，找回目录
