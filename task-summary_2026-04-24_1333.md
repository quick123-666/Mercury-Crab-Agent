# 对话自动保存到知识库

## 目标

实现对话内容自动保存到 llmwikify 知识库。

## 完成的工作

### 1. 修复 Windows GBK 编码 bug
- **问题**: llmwikify 的 `pathlib.Path.write_text()` 和 `read_text()` 在 Windows 上默认使用 GBK 编码，遇 emoji 和中文报 `UnicodeEncodeError`
- **修复**: 在 `C:\Program Files\Python313\Lib\site-packages\llmwikify\core\wiki_mixin_page_io.py` 中，给所有 `write_text()` 和 `read_text()` 调用添加 `encoding='utf-8'` 参数（共 6 处）
- **结果**: 彻底消除编码错误，write_page 正常工作

### 2. 今日对话已写入知识库
- 页面名: `source/conversation-2026-04-24`
- 包含今天所有会话的主题摘要、关键决策和技术笔记
- 搜索功能验证通过

### 3. 周期性自动保存机制
- 配置了 `HEARTBEAT.md`，每次心跳触发时检查是否有新对话需要保存
- 由于 `openclaw cron` CLI 存在 stack overflow 错误（配置循环引用），无法用 cron 任务，改用 heartbeat 方案
- 页面命名规范: `conversation-YYYY-MM-DD`（每天一页，追加内容）
- 页面类型: `source`
